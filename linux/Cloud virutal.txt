1. 클라우드와 가상화 기술 배경 
클라우드 컴퓨팅
- 하드웨어와 이들을 연결하는 네트워크 통합된 계산, 저장 및 처리가 가능한 환경으로 제공
- IT발전으로 하드웨어 성능이 증가하는데, 
  서버가 구동하는애플리케이션의 양이 적어 서버의 성능을 전부 활용하지 못하는 비표율적인 방황이 발생
- 기업들은 가상화를 통해 서버를 통합시킴으로써 구매 비용을 아끼고, 서버 운영 측면에도 이익을 얻음.

2. 가상화 기술 개념
- 가상화 : 물리적인 컴포넌트를 논리적인 객체로 추상화 하는 것
- 하나의 장치를 여러개로 동작 or 여러 개의 장치를 묶어 하나의 장치인 것처럼 사용할 수 있는 기술
- 가상화의 대상이 되는 컴퓨팅 자원을 가상화함으로써 높은 수준의 사용률과 분산 처리 능력을 얻음.

3. 가상화 종류
(1) 서버 가상화
- 가상화를 가능하게 하는 하이퍼바이저(Hypervisor)
- 애플리케이션을 실행하기 위한 컴퓨텅 환성 Virual Machine, VM
- 각각의 가성머신들을 관리할 가상머신모니터(Virual Machine Monitor)
// 하이퍼바이저는 하드웨어의 물리적인 리소스를 VM들에게 제공
   VM과 하드웨어 간의 I/O명령 처리
   Virtualization Layer에 위치하며 하드웨어와 다수의 VM들 사이의 인터페이스라 봄.

(2) 데스크탑 가상화
- 데이터 센터의 서버에서 운영되는 가상의 PC환경
- 물리적으로 존재하지 않는 가상의 개별 컴퓨터로 단순한 인터페이스만으로
  컴퓨터를 활용할 수 있음.
- 네트워크만 연결이 된다면 서버에 접속아형 자신만의 PC환경을 구동시킬 수 있음.
- 복원, 생성 등의 작업이 쉬워지며, 보안 측면에서 데이터 센터급의 서비스를 보장받을 수 있음.

(3) 애플리케이션 가상화
- 운영체제로 부터 응용 소프트웨어를 캡슐화하는 기법
- 윈도우 XP에서만 구동되는 구형 프로그램을 실행하고자 할 때 용이하게 쓰임
- 가상화된 애플리케인션은 관리자가 배포 및 업데이트를 할 때 상대적으로 유용
