리눅스
: 컴퓨터 운영 체제의 하나, 그 커널을 뜻하기도 한다.
: 다중 사용자, 다중작업, 다중 스레드를 지원하는 네트워크 운영 체제(NOS)
-> 리눅스 커널과 GNU 프로젝트의 라이브러리와 도구들이 포함된 
     전체 운영체제(GNU/리눅스)

: 개인용 컴퓨터, 슈퍼컴퓨터, 휴대전화, 스마트TV, 
  개인용 비디오 레코더와 같은 임베디드 시스템까지 이용되고 있다.

: GNU 프로젝트는 리눅스 시스템의 핵심적인 요소의 대부분을 구성한다.
-> 시스템 라이브러리, 컴파일러, 텍스트에디터, 쉘등 핵심적인 부분 완성

리눅스의 특징
1. Linux의 특징 및 기능
2. 리눅스의 신뢰성
: 하드웨어 드라이버 구동을 시스템 차원에서 철저히 분리하여 관리한다.
: 하드웨어 드라이버 접근을 철저히 시스쳄의 감시하여 시스쳄을 완벽하게 보호한다.

리눅스에 관련된 기본상식
: GNU프로젝트를 통해 "완전한 유닉스 호환 소프트웨어 시스템"이 되는 것이 목표
: 자유 소프트웨어 재단(FSF)의 중심이었지만 GNU안정판이 나오지 않게 되지만,
   GNU 시스템은 유닉스와 호환될 수 있도록 만들어졌습니다.
1. GNU 프로젝트
-> 목표 누구나 자유롭게 "실행, 복사, 수정, 배포"할 허가권 아래 소프트웨어를 배포한다.
2. GNU(Gnu's Not Unix)란?
-> 무료로 배포하고 있는 유닉스 운영체제 호환 컴퓨터 프로그램의 총칭
-> 이들의 목적은 바로 프로그램의 공유
-> 소프트웨어를 공유 함을써 사회로 환원된다는 믿고 프로그램을 개발.
3. GPL(General Public License)이란?
-> 자유 소프트웨어 재단에서 만든 자유 소프트웨어 라이센스이다.
(1) 컴퓨터 프로그램을 어떤 목적으로든지 사용할 수 있다.
(2) 프로그램 복사를 언제나 코드와 함께 판매 or 무료로 배포할 수 있다.
(3) 컴퓨터 프로그램의 코드를 용도에 따라 결정할 수 있다.
(4) 프로그램 역시 코드와 함께 자유로이 배포할 수 있다.
4. 오픈소스 [Open source]
-> 일종의 프로그래밍 '설계지도'인 소스코드를 무료 공개, 배포하는 것
-> 자유 소프트웨어 말 대신 오픈소스 소프트웨어라는 표현을 시작한다.
5. 리눅스는 누구 무엇 때문에 만들었을까?
-> 리누스를 POSIX를 도입하여 PC에서 사용할 수 있는 유닉스 버전을 만들기로 결심한다.
6. 리눅스 기본개념
-> 리눅스라는 것은 사실 kernel자체이다.
-> OS중에서 core이며, hardward,process,scheduling,networking 등등
   storage, RAM, CPU,network card을 상ㅇ자에게 서비스 해주는 역할을 담당한다
-> kernel은 서비스이며, 사용자가 서비스를 요청하지 않으면 아무런 일도 하지 못한다.

GNU라는 단체가 open source로 구현하여
kernel 위에 system util들이 존재하여 Linux 명령어들이 있다.
bash는 이러한 명령어를 사용자가 system util들을 호출하고 그 결과 볼수 있도록 제공한다.
// 사용자가 쉽게 하나의 완전한 package로 만들어서 배포하는 것이다.
    특정 package는 단위로 install 하거나 제거가 쉽게 만들어 두고 있다.
