컴퓨터 운영 체제의 하나이며, 그 커널을 뜻하기도 한다.
다중 사용자, 다중작업(멀티태스킹), 다중 스레드를 지원하는 네트워크 운영 체제(NOS)

리눅스 : 리눅스 커널 + GNU 프로젝트의 라이브러리와 도구들이 포함한다.

리눅스의 특징
: 하나 이상의 여러 개의 애플리케이션을 동작시킬 수 있다.
: 하드웨어 드라이버 구동을 시스템 차원에서 정저히 분리하여 관리한다.

리눅스의 신뢰성
: 데이터를 얼마나 신뢰성있게 보호할 수 있는가
: 서버로서 프로그램이 요구하는 기능을 운영체제가 얼마나 신뢰성있게 제공하느냐

리눅스의 기본상식
: GNU -> 무료로 배포하고 있는 유닉스 운영체계 호환 컴퓨터 프로그램
      -> 프로그램의 공유를 위한 유닉스 호환 자유 소프트웨어를 제작해온 단체
: GPL -> 자유 소프트웨어 재단에서 만든 자유 소프트웨어 라이센스
      -> 목적에 부합되어 무료로 이용이 가능하고 용도 맞게 결정 및 배포가 가능하다.
: 오픈소스 -> '설계지도'인 소스코드를 무료 공개, 배포하는 것

리눅스 개본개념
: 리눅스라는 것은 사실 kernel자체이다.
: 운영체제 중의 core,hardware,process,scheduling,networking
  storage, RAM, CPU, network card을 사용자에게 서비스 해주는 역할을 담당
// kernel은 서비스이며 사용자가 서비스를 요청하지 않으며 아무런 일도 하지 못함

리눅스 배포판
: GNU util들을 모아 수정을 통해 사용자가 사용하기 쉽게
  하나의 완전한 package로 만들어서 배포하는 것이다.

Desktop install
1. ppa-purge 
: PPA를 통해 설치한 모든 프로그램 삭제
: 저장소명을 이용한 삭제가 가능합니다.
2. PPA(Personal Package Archive)
: '우분투 소프트웨어 센터'에서 원하는 모든 것들을 검색하고 설치할 수 있음.

우분투 소프트웨어 센터의 소프트웨어들은 안정하고 안정적이라고 판단되는 것들만 모여있다.
최신버전이 나올 경우 우분투의 다음 버전까지 기다려야 하는 상황이 생긴다.
그래서 필요한 기능이 PPA(PErsonal PAckage Archive)입니다.
우분투가 업데이트하는 것보다 더빠르게 업데이트를 제공하므로, 
우분투 소프트웨어 센터에 등록되지 않은 프로그램을 설치하는데 가장 편리하고 좋은 방법이다.
ppa를 통해 삭제하는 번거로움을 해결해주는 것이 'ppa-purge'패키지를 통해 해결할 수 있다.
