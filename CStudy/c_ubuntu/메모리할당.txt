
프로그램 
: 프로그래머가 만든 프로그램 실행 파일

프로세스 
: CPU가 실행 할 수 있도록 명령어를 읽어 메모리에 재구성하는 것
: 실행중인 프로그램
: 세그먼트의 집합으로 구성

세그먼트
: 사용자가 입력한 데이터를 기억하는 메모리 공간
: Code, Data, Stack 세그먼트로 구성되어 있음

전역 변수 : 프로그램 시작~종료까지 유지
지역 변수 : 메모리에 할당 되었다가 함수가 끝나면 사라짐

스택
: BP를 기준의 데이터타입의 공간만큼 SP로 데이터를 쌓고 빼는 활동
: 실제 메모리에서는 데이터 공간이 추가되면 스택포인터-데이터타입.
: main함수BP + IP + BP + Test Function [BP+SP]
  스택프레임에서 빠질 때는 쌓아놓은 순서 반대로 으앙 된다.

스택프레임
: 함수를 호출할 때 일어나는 스택의 변수 -> C언어 코드를 기계어로 번역하는 시점에 결정.


메모리할당 : 데이터를 저장할 공간을 적정하게 나누는 작업
1. 정적 메모리 할당
-> 프로그램 실행 중에 메모리 크기 변경불가
-> 다시 컴파일 하지 않도록 크기에 맞는 변수로 할당하자
-> 프로그램 실행시 메모리의 위치가 결정됨.
-> 프로그램 실행 중에 할당된 메모리의 크기는 변경 불가

- 정적으로 할당된 메모리 관리하는 법
1. 그 변수의 현재 주소를 알아야 한다.
2. 주소를 기억하기 위해 변수의 개수만큼 메모리가 필요

2. 동적 메모리 할당
- 스택에 비해 큰 크기의 메모리 할당하닝
- 메모리를 할당하고 해제하는 시점을 지정 가능
- 프로그램 실행 중에 자동으로 메모리 크기 변경가능
